메타데이터의 역할
----------------
* 데이터를 설명하는 데이터
* 검색 엔진에서 문서 정보를 용이하게 수집하고 분류하게 도움을 주기 위한 정보

```<title>```
-----------------

* 브라우저의 제목 표시줄이나 페이지 탭에 보이는 문서 제목을 정의
* 문자열만 작성가능
* <head>태그 안에 하나의 <title>태그만 작성가능
* 페이지 제목은 페이지를 간략하게 설명하는 내용을 적는것이 좋음
  
※명사로만 작성하게 되면 검색엔진에서 순서를 내리곤 한다.

```<meta>```
------------------
속성
*  name : name과 content 특성을 함께 사용하면 문서의 메타데이터를 이름-값 쌍으로 제공할 수 있다. 
```
HTML에서 정의하는 표준 메타데이터 이름

* application-name:웹 페이지에서 구동중인 애플리케이션의 이름
* author: 문서 저작자
* description:페이지에 대한 짧고 정확한 요약 특정 브라우저에서 즐겨찾기 페이지의 기본 설명값으로 description을 사용한다.
* generator: 소프트웨어를 통해 페이지를 만들었을때  페이지를 생성한 소프트웨어의 식별자
* keyword:페이지의 콘텐츠와 관련된, 쉼표로 구분한 키워드 목록 인스타그램의 해쉬태그와 같은 용도
* referrer: 문서에서 시작하는 요청의 HTTP Referer 헤더를 통제
* viewport: device장치에서 보이는 웹 브라우저 화면의 너비, 비율 등등 설정할 수 있다.
```
* charset : 페이지의 문자 인코딩을 선언. 이 특성이 존재할 경우, 그 값은 반드시 문자열 "UTF-8" 대소문자 구분 없어야 하고,
  <title>전에 작성하거나 <head>태그의 첫번째 줄에 명시하는걸 권장함

* MIME타입 :  외부 파일의 확장자를 못 읽는 경우를 위해서 type을 명시하는것

일반적인 구조 ``` type/확장자```
    
```<style>```
-----------------
* 문서나 문서 일부에 대한 스타일 정보를 나타내는 요소
* 간단한 스타일 정보 같은 경우는 <head>태그 안에 작성해도 되지만 일반적으로 외부css 파일을 link로 연결하는것을 권장
* 두개의 <style>요소가 있을때 두 요소에 같은 요소의 스타일정보가 있는경우에는 뒤쪽<style>이 앞쪽을 덮어쓴다.    

```<script>```
-----------------
*  <style>태그처럼 인라인으로 직접 스크립트를 작성할 수도있고, 외부의 스크립트파일도 불러올 수 있다. 보통 JavaScript파일과 함께 쓴다.

* 웹 브라우저는 가장 위에있는 코드부터 아래로 해석 하기때문에  <head>태그나 <body>태그 중간에 <script>태그가 있으면 브라우저가 화면을 렌더링 하는중에 <script>태그를 해석해버리기 때문에 <script>태그는 화면이 다 렌더링이 되고 실행이 되도록 <body>태그 맨 아래에 작성하는 것을 권장한다.
