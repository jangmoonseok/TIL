```<div>```, ```<span>```
------------------
* ```<div>```
  * 구획을 나타내는 통용 컨테이너 역할
  * 순수 컨테이너로서 아무런 의미를 표현하지 않고, 다른 요소 여럿을 묶어 CSS스타일링을 돕기위해 사용됨
  * Block요소
  
* ```<span>```
  * 구문 콘텐츠를 위한 통용 인라인 컨테이너 역할
  * 순수 컨테이너로서 텍스트의 CSS스타일링을 돕기위해 사용됨

시멘틱 요소
-----------------
* ```<div>```와 달리 의미를 가지고있는 컨테이너 요소
* 각 요소의 의미를 고려하여 구조를 설계하고, 코드를 작성하는것을 권장

### 장점
* 검색엔진이 페이지를 분석할때 중요 키워드로 간주해 검색 랭킹에 영향을 줄 수 있다.
* 시작 장애 사용자가 스크린리더를 사용 할때 푯말로 사용돼 웹 접근성이 좋아진다.
* 다른사람이 코드를 볼때 이 컨테이너가 어떤 의미인지 한눈에 알아볼 수 있다.

```<header>```
* 소개 및 탐색에 도움을 주는 컨텐츠를 나타낸다(제목, 로고, 검색, 작성자 이름 등)
* 페이지 제목으로서 사용할땐 한 웹페이지에 한개만 사용하는것을 권장
* 헤더안에 또 다른 헤더와 푸터를 사용할 수 없다.

```<footer>```
* 구획의 작성자, 저작권 정보, 관련 문서 등의 내용을 담는다.
* 웹 페이지의 가장 아래에 작성하는것이 일반적이다.
* 푸터안에 또 다른 푸터와 헤더를 사용할 수 없다.

```<nav>```
* 현재 페이지가 어딘지 또는 다른 페이지로의 링크를 보여주는 구획을 나타낸다. 메뉴,목차,색인으로 자주 사용됨

```<aside>```
* 문서의 주요 내용과 간접적으로 연관된 부분을 나타낸다.
* 주로 사이드바 혹은 콜아웃 박스로 표현된다.
* 다른 페이지로 이동하는 링크를 넣는 용도인```<nav>``` 와 달리  추가적인 정보같은 본문 내용과 관련이 있는 정보를 담기때문에 없어도 크게 문제없다. 

```<main>```
* body 내에서 주요 컨텐츠의 부모 컨테이너 역할
* header나 footer와 같이 하나 이상의 태그는 사용하면 안된다.
* IE브라우저에서는 호환이 되지않아서
```<main role="main">``` 이런식으로 태그를 작성해야한다. 

```<article>```
* 독립적으로 구분해 배포하거나 재사용을 하더라도 의미적으로 문제가 없는 구획을 나타낸다
* 게시판과 블로그 글, 매거진, 뉴스 기사 등을 주로 나타낸다.
* 하나의 문서에 여러 개 가질 수 있고, 블로그의 경우 각각의 글이 ```<article>```요소가 되며,그 안에 여러 개의 ```<section>```을 가질 수 있다.
* ```<article>```자체로 독립적인 컨텐츠가 될 수 있기 때문에```<article>```을 식별할 수단으로 주로 제목 요소를 포함한다. 

```<section>```
* HTML 문서의 독립적인 구획을 나타내며, 더 적합한 의미를 가진 요소가 없을 때 사용
* 이 구획이 독립적으로 재사용될  때에도 의미를 이해할수 있으면 ```<article>``` 그게 아니라면 ```<section>```
* 마찬가지로 각각의 ```<section>```을 식별할 수단으로 주로 제목요소를 자식으로 포함한다.
