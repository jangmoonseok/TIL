```<div>```, ```<span>```
------------------
* ```<div>```
  * 구획을 나타내는 통용 컨테이너 역할
  * 순수 컨테이너로서 아무런 의미를 표현하지 않고, 다른 요소 여럿을 묶어 CSS스타일링을 돕기위해 사용됨
  * Block요소
  
* ```<span>```
  * 구문 콘텐츠를 위한 통용 인라인 컨테이너 역할
  * 순수 컨테이너로서 텍스트의 CSS스타일링을 돕기위해 사용됨

시멘틱 요소
-----------------
* ```<div>```와 달리 의미를 가지고있는 컨테이너 요소
* 각 요소의 의미를 고려하여 구조를 설계하고, 코드를 작성하는것을 권장

### 장점
* 검색엔진이 페이지를 분석할때 중요 키워드로 간주해 검색 랭킹에 영향을 줄 수 있다.
* 시작 장애 사용자가 스크린리더를 사용 할때 푯말로 사용돼 웹 접근성이 좋아진다.
* 다른사람이 코드를 볼때 이 컨테이너가 어떤 의미인지 한눈에 알아볼 수 있다.

```<header>```
* 소개 및 탐색에 도움을 주는 컨텐츠를 나타낸다(제목, 로고, 검색, 작성자 이름 등)
* 페이지 제목으로서 사용할땐 한 웹페이지에 한개만 사용하는것을 권장
* 헤더안에 또 다른 헤더와 푸터를 사용할 수 없다.

```<footer>```
* 구획의 작성자, 저작권 정보, 관련 문서 등의 내용을 담는다.
* 웹 페이지의 가장 아래에 작성하는것이 일반적이다.
* 푸터안에 또 다른 푸터와 헤더를 사용할 수 없다.
