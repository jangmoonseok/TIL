# Form요소

* 정보를 제출하기 위한 대화형 컨트롤 요소

### 속성
* action : 폼 데이터를 전송할 url 주소를 적는속성
* method : 양식을 제출할때 사용할 HTTP메서드
  * get : 폼 데이터를 get방식으로 제출한다.
  get 방식으로 제출한 데이터가 action URL 뒤에 ? 구분자 뒤에 이어 붙여진다.
  * post :  폼 데이터를 post방식으로 제출한다.
  post 방식으로 제출한 데이터는 정보를 요청한 웹 페이지의 서버로 전송된다.
  
  ### 자식요소
 ```<lable>``` : 폼 컨트롤 요소의 항목을 설명하는 태그, 보통 ```<input>``` 앞에 붙어서 사용된다.
  * 속성
    * for : 여러개의 레이블 요소가 있을때 레이블 요소의 id값을 for 의 값으로 넣어 본인의 짝을 찾을때 사용한다.
     ```<lable>```의 자식요소로 ```<input>```을 넣을땐 for 속성을 사용할 필요없다.

```<fieldset>``` : 웹 양식의 여러 컨트롤과 레이블을 묶을 때 사용한다.  ```<legend>```를 첫번째 자식으로 넣어 묶은 그룹의 콘텐츠를 설명한다.

```<input>``` : 웹 기반 양식으로 사용자의 데이터를 받을 수 있는 대화형 컨트롤 요소
```
 자주사용되는 유형
 
 텍스트 필드
  * text : 일반 텍스트를 입력하는 필드생성
  * password : 비밀번호를 입력하는 필드를 생성, 입력한 데이터는 마스킹처리된다.
  * email : 이메일을 입력하는 필드, 이메일 형식이 아닌 데이터를 제출하면 유효성검사를 통해 제출되지 않는다.
 
 숫자 필드
  * tel : 전화번호를 입력하는 필드생성
  * number : 숫자를 입력하는 컨트롤생성
  * range : 숫자가 보이지 않는 범위 위젯으로 나타나는 숫자입력 컨트롤 생성
  * date : 날짜(시간을 제외한 연,월,일)을 입력하는 컨트롤 생성
 
 그 외
  * submit : 폼에 입력된 데이터를 action URL로 전송하는 버튼생성
  * reset : 폼의 내용을 기본 값으로 초기화 하는 버튼생성
  * checkbox : 값을 선택하거나 해제하는 체크박스 생성, checked속성을 사용하여 기본값으로 체크되있도록 할 수 있다.
  * radio : 같은 name을 가진 여러개의 하나의 값만 선택하게 하는 라디오 버튼 생성, 마찬가지로 checked속성 사용가능
```
  * 속성
    * name : input 양식 컨트롤의 이름,역할로 value와 짝을 이루어져 전송된다.
    * placeholder : 양식 컨트롤이 비어있는 때 양식 컨트롤에 나타나는 내용
    * autocomplete : 자동완성 기능 생성
    * required : 불리언 양식으로 필수입력값을 지정한다.
    * disabled : 폼 컨트롤을 비활성화 시킴
    * readonly : 폼 컨트롤을 수정을 불가능하게 하고 읽기만 가능하게 함
    * min, max : 숫자를 입력하는 폼 컨트롤의 범위를 최소,최대로 제한한다.
    * step :  숫자입력 폼의 값을 증가시킬때 step의 값만큼 증가한다.
  * ```<input>```과 ```<button>``` 의 차이점 :
    input은 빈 요소이어서 자식 요소 없이 value값을 문자열로만 작성해 버튼의 값을 넣을수 있고,
    button은 자식을 가지는 태그로 자식요소를 가질수 있어서 다양한 HTML 요소를 넣을수 있다.
