HTML과 웹 브라우저
=================
```
.pdf -> PDF뷰어
.doc, .docx -> MS Word
.html -> 웹브라우저(크롬, 엣지, 익스플로어, 사파리 ...등)

웹 브라우저는 단순히 html문서를 보여주기 뿐 아니라 사용자 입장에서 다양한 편리한 기능과 
개발자입장에서는 웹페이지 분석을 도와주는 개발도구 역할도 한다
```

HTML, CSS, JavaScript
====================
```
HTML: 웹 문서의 기본적인 골격을 담당 (ex:텍스트 입력창을 생성)
CSS: 각 요소들의 레이아웃, 스타일링 담당(ex:텍스트 입력창의 테두리 색을 변경)
JavsScript: 동적인 요소(사용자와의 인터렉션)을 담당
(ex:사용자가 텍스트 입력창에 한 동적행동에 대해 새로운요소를 보이거나, 없애거나, 움직이게 하는 기능)

HTML과 CSS를 나누어쓰면 장점:
  웹에 보여지는 화면과 모바일에 보여지는 화면에서 동일한 구조의 html파일에 각각의 다른 CSS파일을 적용하면
  두개의 html파일을 만들필요없이 스타일링가능
```

웹 표준과 웹 접근성, 웹 호환성
======================
```
1.웹 표준
  브라우저를 만든 회사들이 다르기 때문에 각 브라우저마다 우리가 생각한 의도와 다른모습으로 나올수있기때문에
  회사들끼리 한 약속을 웹표준 이라고한다. 흔히 사람들이 말하는 웹 표준은 HTML5

2.웹 접근성
  어떤 조건의 상황이나 장애가있는 사용자에게도 동일한 접근성을 주기위한 방식
  (스크린리더, 화면 돋보기, 음성 인식, 키보드 오버레이 등)
 ```
