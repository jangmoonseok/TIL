# 선택자의 종류

## 주요 선택자

### 1. Type Selector(태그선택자, 유형선택자)
* 태그의 Type을 선택자로 CSS속성을 적용하는 방식
* 특정 요소를 위해서 Type Selector를 사용하지 않고 일관성있는 스타일링을 위해 공통적으로 사용할 코드를 작성할때 사용하는것을 권장
```
h2{
    color:purple
}
```
* 모든 h2태그의 color를 purple로 적용


### 2. ID Selector
* html태그의 id속성의 값을 선택자로 CSS속성을 적용하는 방식
* 동일한 id속성 값을 가지면 안되고 유일해야한다.

```
#welcome-title {
  color: red;
}
```
* welcome-title이라는 id값을 가진 요소의 color를 red로 적용


### 3. Class Selector
* html태그의 class 속성의 값을 선택자로 CSS를 적용하는 방식
* ID Selector와 달리 여러개의 동일한 Class 값을 가져도 된다.
* 스페이싱을 사용해 두가지 이상의 Class 값을 가질수 있다.

```
.blue {
  color: blue;
}

.red {
  color: red;
}
```
* blue, red라는 class값을 가진 요소의 color를 blue,red로 적용
* 스페이싱을 이용해 blue red 라는 class값을 가진 요소는 뒤에 적용한 red가 덮어씌운다.


## 속성 선택자

### 1.[attr]
* 태그의 속성이름을 선택자로 해당 속성 이름을 가진 태그에 CSS를 적용하는 방식


```
a[target] {
  color: red;
}
```
* a태그의 target속성을 가진 모든 요소의 color를 red로 적용

### 2.[attr=value]
* 태그의 속성이름과 값을 선택자로 해당 속성-값을 가진 태그에 CSS를 적용하는 방식


```
a[href="https://example.org"]
{
  color: purple;
}
```
* a태그의 href속성을 가진 애들중 값이 "https://example.org"인 요소의 color를 purple로 적용


### 3. [attr^=value]
* 태그의 속성-값을 가진 애들중 값이 value로 시작하는 애들 모두를 선택자로 CSS를 적용하는 방식
```
1. a[href^="http"] {
  color: red;
}


2. a[href^="https"] {
  color: green;
}

```
* 1. a태그의 href속성을 가진 애들중 값이 http로 시작하는 요소의 color를 red로 적용
* 2. a태그의 href속성을 가진 애들중 값이 https로 시작하는 요소의 color를 green으로 적용

### 4. [attr$=value]
* att^와 반대로 태그의 속성-값을 가진 애들중 값이 value로 끝나는 애들 모두를 선택자로 CSS를 적용하는 방식
```
a[href$=".com"] {
  color: aqua;
}
```
* a태그의 href라는 속성을 가진 애들중 값이 .com으로 끝나는 요소의 color를 aqua로 적용

### 5. [attr*=value]
* 태그의 속성-값을 가진 애들중 값에 value라는 문자열을 하나라도 가진 모든 애들을 선택자로 CSS를 적용하는 방식
```
a[href*="example"] {
  color: brown;
}
```
* a태그의 href라는 속성을 가진 애들중 값에 example이라는 문자열을 가진 요소의 color를 brown으로 적용


## 가상 클래스 선택자
* 요소의 상태나 몇번째 자식인지 등등 좀더 좁은 범위를 선택자로 CSS를 적용하는 방식

### 1. :first-child
* 요소의 형제들중 첫번째 자식을 선택자로 CSS를 적용한다.
* 해당 태그나, 선택자를가진 첫번째 자식이 없으면 적용되지 않는다.
```
1. li:first-child {
  color: red;
}


2. .Movie:first-child {
  font-size: 24px;
}

```
* 1. li태그의 형제들 중에 첫번째 자식인 li태그의 color를 red로 적용
* 2. Movie라는 클래스값을 가진 요소의 형제들중 첫번째 자식인 .Movie의 font-size를 24px로 적용

### 2.  :last-child
```
span:last-child {
  color: aqua;
}
```
* span태그의 형제들중에 마지막 span태그의 color를 aqua로 적용


### 3. :nth-child
```
1. li:nth-child(2) {
  color: pink;
}


2. li:nth-child(2n -1) {
  color: pink;
}


3. li:nth-child(even) {
  color: pink;
}

```
* 1. li태그의 형제들중 2번째 자식인 li태그의 color를 pink로 적용
* 2. li태그의 형제들중 2n-1의 값을 만족하는 자식인 li태그의 color를 pink로 적용
* 3. li태그의 형제들중 짝수번째 자식인 li태그의 color를 pink로 적용


### 4.  :first-of-type
* 특정타입을 가진 형제들중에 첫번째 자식을 선택
```
1. .movie:first-of-type {
  color: red;
}

2. .movie:first-child {
  color: aqua;
}

```
* 1. movie클래스를 가진 요소들 중에 특정 타입을 가진 첫번째 자식의 color를 red로 적용
* 2. movie클래스를 가진 요소들 중에 첫번째 자식의 color를 aqua로 적용


### 5. :last-of-type, nth-of-type
* first-child vs first-of-type 의 차이와 같은 원리로 작용된다.

### 6. :not
* selector:not(selector) 로 작성하며 첫번째 selector를 선택하고 그 중에 두번째 selector를 제외한 애들을 선택한다.

```
1. input:not(.pw) {
  background-color: aqua;
}

2. input:not([placeholder]) {
  background-color: yellow;
}

3. input:not([type="submit"]) {
  background-color: red;
}
```
* 1. input태그중 pw클래스선택자를 가진 요소를 제외한 모든 요소의 background-color를 aqua로 적용
* 2. input태그중 placeholder라는 속성을 가진 요소를 제외한 모든 요소의 background-color를 yellow로 적용
* 3. input 태그중 type="submit"이라는 속성을 가진 요소를 제외한 모든 요소의 background-color를 red로 적용

### 7. :link, :visited
* a태그의 상태에 따라 CSS스타일링을 적용
```
1. a:link {
  color: brown;
}

2. a:visited {
  color: yellowgreen;
}
```
* 1. 방문하지 않은상태의 a태그의 color를 brown으로 적용
* 2. 방문한 상태의 a태그의 color를 yellowgreen으로 적용

### 8. :hover, :active, :focus
* hover : 요소에 마우스를 올린 상태일때만 CSS적용
* active : 요소를 마우스로 클릭한(누르고있는) 상태일때만 CSS적용(link,visited,hover보다 앞에 적용했을때 덮어씌워진다)
* focus : 요소에 포커싱이 돼있을때 CSS적용
```
1. input[type="button"]:hover {
  color: white;
}

2. input[type="button"]:active {
  background-color: red;
  color: white;
}

3. input[type="text"]:focus {
  background-color: blue;
  color: white;
}
```

### 9. :enabled, :disabled, :checked
* input요소에서 자주 사용된다.
* enabled : input요소의 속성이 활성화 돼있는 disabled가 아닐때 CSS적용
* disabled : input요소의 속성이 disabled일때만 CSS적용 
* input요소의 속성이 checked일때만 CSS적용

```
input[type=text]:enabled{
  background-color: blue;
}

input[type=text]:disabled{
  background-color: aqua;
}

input[type=radio]:checked{
  border: 1px solid red;
}

```

## 가상 요소 선택자
* 실제로 존재하지 않는 요소를 만들거나 범위를 선택해 CSS를 적용
* 실제로 존재하는 콘텐츠에 사용하지않고 꾸밈을 위해 사용

### 1. ::before, ::after
* 실제로 존재하지 않는 가상요소를 만들어 앞, 뒤에 붙인다.

```
.favorite::before{
  content: 'Favorite';
}
```
* favorite이라는 클래스를 가진 요소 앞에 'Favorite'이라는 가상요소를 붙인다.

### 2. ::first-letter, ::first-line, ::selection
* first-letter : 요소의 컨텐츠의 첫번째 글자에만 CSS적용됨
* first-line : 요소의 컨텐츠의 첫번째 줄에만 CSS 적용됨
* selection : 요소의 컨텐츠의 선택영역에만 CSS적용됨

## 선택자 결합

### 1. 하위 선택자 결합
* 스페이싱을 이용해서 두가지 이상의 선택자를 결합해 앞쪽에 있는 요소들로 범위가 좁혀진다.

```
ul li:last-of-type {
  color: pink;
}
```
* ul 태그에 속한 li태그 중에 마지막에 위치한 li태그의 color를 pink로 적용

### 2. 자식 선택자 결합
* > 를 사용해서 앞쪽에있는 선택자의 직속 자식중에서 탐색을한다.

```
ul > li:last-of-type {
  color: pink;
}
```
* ul태그의 직속 자식의 li태그중에 마지막에 위치한 list 태그의 color를 pink로 적용

### 3. 일반 형제 선택자 결합(~)
* selector ~ selector 로 작성하며 두 요소는 서로 형제관계 이어야 한다.
* 뒤에오는 selector는 앞에있는 selector보다 밑에 작성돼있어야한다.
```
span ~ p {
  color: red;
}
```
* span태그 기준으로 밑에있는 모든 형제 p태그의 color를 red로 적용

### 4. 인접 형제 선택자 결합(+)
* selector + selector로 작성하며 두 요소는 서로 형제관계 이어야 한다.
* 앞에오는 selector와 뒤에오는 selector가 인접해야만 CSS가 적용된다.
* 뒤에오는 selector는 앞에있는 selector보다 밑에 작성돼있어야한다.
```
.red + p {
  color: blue;
}
```
* red라는 클래스를 가진 요소와 인접하면서 밑에 있는 p태그의 color를 blue로 적용

### 5. 그룹화
* 여러가지 요소들을 공통적인 스타일링을 하고 싶을때 , 를 이용해 그룹화한다.
```
p,
span,
div {
  background-color: yellowgreen;
}
```
## 범용 선택자
* 문서의 모든 요소에 CSS를 적용한다.
* CSS파일 맨위에 작성하는것을 권장한다.
* 다른 선택자들과 같이 결합해서 사용도 가능하다.
```
div > * {
  background-color: antiquewhite;
}
```
div의 자식들 모두 background-color를 antiquewhite로 적용

## 상속 제어

### 1. initial : 부모의 스타일을 상속받고 싶지 않을때 사용
```
.child2 {
  color: initial;
}
```
* chile2라는 클래스를 가진 요소는 부모로부터 color속성을 상속받지 않는다.

### 2. inherit : 부모의 스타일을 무조건 상속시키고 싶을때 사용
```
.parent1,
.parent2 {
  color: blue;
}

.child2 {
  color: red;
}
.parent2 * {
  color: inherit;
}
```
* child2라는 클래스를 가진 요소의 color를 red로 지정을 했지만 parent2라는 클래스를 가진 요소의 모든 요소들의 color를 inherit으로 적용했기때문에 부모요소인 parent2의 color:blue가 적용된다.

### 3. unset
* 부모로부터 상속받을 값이 있을 때 : inherit
* 부모로부터 상속받을 값이 없을 때 : initial
