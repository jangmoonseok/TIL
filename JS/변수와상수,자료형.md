# 변수와 상수

## 변수란?
* 변경 가능한 값을 저장하기 위한 기억공간
* 사용하기 전 반드시 선언해야함
* 변수명은 식별자를 이용해 정해야함
* 중복 선언 불가능
* 키워드:let


![image](https://user-images.githubusercontent.com/64582209/134796764-65a7ba28-e657-41a3-8e4e-0e13ec4692d7.png)


## 상수란?
* 변경 불가능한 값을 저장하기 위한 기억공간
* 사용하기 전 반드시 선언해야함
* 상수명은 식별자를 이용해 정해야함
* 상수명은 대문자로 표기하는것을 권장
* 중복 선언 불가능
* 선언과 동시에 값이 초기화
* 키워드:const

![image](https://user-images.githubusercontent.com/64582209/134796951-48bb1660-b594-4070-94ce-c7e2d576b88e.png)

### 식별자
* 스크립트에서 변수나 함수에 이름을 붙일 때 사용
* 대소문자를 구별하며 유니코드 문자셋을 이용
* 자바스크립트 내 식별자 규칙
  1. 키워드 사용 불가
  2. 숫자로 시작 불가
  3. 특수문자는 _와$만 허용
  4. 공백 문자 포함 불가

### 변수와상수 예제코드
```
// 선언 후 할당
let hi;
hi = "hello"
// 선언과 동시에 초기화
let halo = "hello!"

// 한 줄에 여러 변수 선언과 초기화
let name = "john", age = 12, msg = 'hello!!'

// 상수는 대문자로 표기
const TESTCASE = 5

console.log(hi) //output:hello
console.log(halo)//output:hello!
console.log(name)//output:john
console.log(TESTCASE)//output:5
```

### 호이스팅
* 코드에 선언된 변수 및 함수를 유효한 범위의 코드 상단으로 끌어 올리는 작업
* var의 변수/함수의 선언만 위로 올려지고, 할당은 올려지지 않음
* let/const 변수 선언과 함수 표현식에서는 호이스팅 발생하지 않음

```
// using var

console.log(name_1);//output:undefined
// var를 사용하면 선언부분만 호이스팅되고 초기화는 되지않음
var name_1 = "john"
console.log(name_1)//output:john

// using let
console.log(name_2);//초기화오류
// let을 사용하면 아예 호이스팅이 되지않아 오류가뜬다.
let name_2 = "john";
console.log(name_2)//output:john
```

# 자료형
* 목적에 따라 특별한 성질이나 정해진 범주를 갖고 있는 데이터의 종류
* 6가지의 원시 타입 자료형과 1가지의 객체 타입 자료형으로 구성

![image](https://user-images.githubusercontent.com/64582209/134797597-b5ba1652-abea-4863-a9bc-5a51fe940c0c.png)

## 원시타입
### 1. typeof
* typeof는 인수의 자료형을 반환하는 연산자
* 연산자인 typeof x와 함수인 typeof(x)로 문법 지원
### 2. boolean
* boolean은 논리적인 값을 표현하는 자료형
* 참인 true와 거짓인 false, 두 가지 값만 존재
* 주로 조건문 등에서 동작 판단의 기준으로 사용

### 3. null & undefined
* null
  * null은 값이 비어 있다는 의미로 표현되는 자료형
  * 존재하지 않는, 비어있는, 알 수 없는 값을 나타내는데 사용

* undefined
  * undefined는 값이 할당되어 있지 않은 상태를 나타낼 때 사용되는 자료형
  * 변수 선언 후 초기화 하지 않는다면, undefined가 자동으로 할당
